## 1. 專案概述

### 1.1 背景與動機

在舉辦各類活動（如聚餐、團建、研討會等）時，主辦方需要一個簡單且有效的方式來收集參與者的報名資訊，包括出席意願、餐飲需求等。傳統的紙本報名或散落的訊息溝通方式容易遺漏資訊、難以統計。本系統旨在提供一個**線上活動報名表單**，讓參與者能快速填寫資料，同時讓管理者能在後台集中檢視與管理所有報名資訊。

### 1.2 目標用戶

| 角色 | 描述 |
|------|------|
| **一般用戶（報名者）** | 不限身份，任何想參與活動的人皆可填寫報名表單 |
| **管理者（主辦方）** | 擁有後台管理權限，可檢視、編輯、管理所有報名資料 |

### 1.3 核心價值主張

- **簡單易用**：報名者無須註冊帳號，直接填寫表單即可完成報名
- **資料集中**：所有報名資訊統一收集在後台，方便管理者快速查看與統計
- **彈性管理**：管理者可透過後台修改參與者資料，應對臨時變動

## 2. 功能需求

### 功能一：報名表單填寫

> 作為**一般用戶**，我希望能**填寫一份報名表單，輸入姓名、出席意願、餐盒需求、葷素選擇及備註**，以便**主辦方了解我的參加意願與需求**。

**表單欄位說明：**

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| 姓名 | 文字輸入 | ✅ | 報名者姓名 |
| 是否參加 | 單選（是／否） | ✅ | 是否有意願參加活動 |
| 是否需要餐盒 | 單選（是／否） | 條件必填 | 僅當「是否參加」選擇「是」時顯示 |
| 葷／素 | 單選（葷／素） | 條件必填 | 僅當「是否需要餐盒」選擇「是」時顯示 |
| 備註 | 多行文字輸入 | ❌ | 自由填寫其他意見或需求 |

### 功能二：後台資料總覽

> 作為**管理者**，我希望能**在後台檢視所有報名者的資料列表**，以便**快速掌握報名狀況與統計人數**。

- 以表格形式展示所有報名紀錄
- 顯示欄位包含：姓名、是否參加、是否需要餐盒、葷素選擇、備註、提交時間
- 支援基本統計：參加人數、餐盒數量、葷素比例

### 功能三：後台資料編輯（需管理者權限）

> 作為**管理者**，我希望能**修改任一報名者的資料**，以便**在參與者臨時變更需求時及時更新紀錄**。

- 管理者可點選任一筆資料進行編輯
- 編輯操作需登入管理者帳號後才可使用
- 修改後系統記錄更新時間

### 功能四：管理者登入驗證

> 作為**管理者**，我希望能**透過帳號密碼登入後台**，以便**確保只有授權人員可以存取與修改報名資料**。

- 提供管理者登入頁面
- 帳號密碼驗證
- 登入後方可進入後台管理頁面
- 支援登出功能

### 功能五：報名成功回饋

> 作為**一般用戶**，我希望**在提交表單後看到明確的成功提示**，以便**確認我的報名已被系統接收**。

- 表單提交後顯示成功訊息
- 顯示已填寫的摘要資訊供確認

### 功能六：報名資料刪除

> 作為**管理者**，我希望能**刪除錯誤或重複的報名紀錄**，以便**維持資料的正確性**。

- 管理者可在後台刪除指定報名紀錄
- 刪除前需確認提示，避免誤刪

### 功能七：資料匯出

> 作為**管理者**，我希望能**將報名資料匯出為 CSV 檔案**，以便**用於其他用途（如製作名牌、訂餐統計等）**。

- 後台提供匯出按鈕
- 匯出格式為 CSV，包含所有欄位

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 技術選擇 |
|------|----------|
| 後端框架 | Flask (Python) |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML + CSS + JavaScript |

### 3.2 效能考量

- 系統預期為小型活動使用，同時在線人數不超過 50 人
- 頁面載入時間應控制在 2 秒以內
- SQLite 足以應付預期的資料量

### 3.3 安全考量

- 管理者密碼需使用雜湊（hash）儲存，不可明文存放
- 後台頁面需進行身份驗證，未登入者不可存取
- 表單輸入需進行基本驗證與防止 XSS 攻擊

### 3.4 可用性考量

- 表單頁面需支援手機瀏覽（RWD 響應式設計）
- 介面需直覺易用，無需說明即可完成填寫
